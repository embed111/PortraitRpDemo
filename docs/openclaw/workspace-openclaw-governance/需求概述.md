# 需求概述

## 1. 项目背景与问题定义
当前 `PortraitRpDemo` 是极简演示型工作区，顶层只有 `AGENTS.md` 和发布有效性标记文件，尚未具备 OpenClaw 风格 `.codex/` 记忆层。

## 2. 目标用户与核心场景
1. 目标用户：演示 agent、owner、执行维护者。
2. 核心场景：
   - 保持 demo 工作区足够轻量；
   - 补齐基础身份、用户与记忆能力。

## 3. 业务目标与成功指标
1. 业务目标：以最小改动补齐 `.codex/`。
2. 成功指标：
   - `.codex/` 核心文档与 `memory/` 已存在；
   - `AGENTS.md` 已具备读取顺序；
   - 现有演示语义不变。

## 4. 范围界定
### In Scope
1. 新增 `.codex/` 和记忆层。
2. 更新 `AGENTS.md`。

### Out of Scope
1. 发布判定逻辑调整。

## 5. 优先级与阶段规划
1. P0：补 `.codex/`。
2. P1：补 `AGENTS.md` 读取顺序。

## 6. 风险、依赖与待确认事项
1. 风险：demo 工作区被过度复杂化。
2. 依赖：执行方按最小模板补齐。
